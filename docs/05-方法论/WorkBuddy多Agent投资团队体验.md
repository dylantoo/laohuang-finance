# 腾讯WorkBuddy多Agent投资团队体验

## 一句话总结

腾讯WorkBuddy是OpenClaw之后第一个把多Agent协同落到产品层面的工具——不是几个API拼接，而是多个Agent（技术分析、财务分析、行业分析、风险评估）真正同时运行、各自独立执行、最后汇总成结构化报告。

---

## 核心内容

### 1. 多Agent协同 ≠ 多角色对话

| 方式 | 描述 | 本质 |
|:-----|:-----|:-----|
| 传统AI | 一个对话框，切换角色 | 单线程串行 |
| 多Agent协同 | 多个Agent同时运行，各干各的 | 多线程并行 |
| WorkBuddy | Agent各自独立执行，最后汇总 | 真正的分工协作 |

关键区别：**不是在同一个对话框里切几个角色来回切换，而是真正的多Agent同时运行。**

### 2. 实测场景：小米财报分析

「股票投资天团」启用了4个Agent：

| Agent | 职责 | 产出 |
|:------|:-----|:-----|
| 技术分析 | MACD、RSI、KDJ量价分析 | 量价趋势判断 |
| 财务分析 | ROE、PE估值数据 | 估值评估 |
| 行业分析 | 竞争格局、行业对比 | 行业定位 |
| 风险评估 | 政策风险、汇率风险、竞争格局变化 | 风险梳理 |

最终输出：**结构化HTML报告**，分板块分视角，有数据支撑有图表呈现。

### 3. 实时更新能力

小米Q1财报出来后重新跑分析：
- SU7交付量增长，但整体EPS下降37%
- 判断从「买入」调到「持有」
- 目标价30.50-31.50
- 提醒Q2财报（8月）才是关键验证点

**不是知识库截止日期的免责声明，而是真正读取最新信息给出更新判断。**

### 4. 范式转变

> 以前面对复杂问题，你会想「我该怎么分析这个」。现在你会想「我该组一个什么样的团队来分析这个」。
>
> 这不是效率变化，这是**组织方式的变化**。

## 对老黄的启示

### 可借鉴的思路

1. **多Agent分工 = 我们的多技能体系** — Hermes的skill体系本质上就是分工：stock-pivot管交易框架、individual-stock-analysis管个股流程、investment-framework管六步总纲。WorkBuddy把这种分工可视化成了并行Agent。
2. **结构化报告 = 我们的简报体系** — 早报/盘中/复盘三时段，本质也是不同时段的Agent各自产出。
3. **实时更新 = 我们的新浪HQ+Tushare实时数据** — 我们已经在做实时数据驱动的分析更新。

### 差距/改进空间

1. **并行度** — WorkBuddy是真正并行，我们的简报是串行生成。理论上可以用delegate_task实现并行采集+汇总。
2. **输出形式** — WorkBuddy输出HTML可视化报告，我们目前是Markdown+PDF。可以考虑生成HTML交互式报告。
3. **自动更新** — WorkBuddy在财报出来后自动重跑，我们需要手动触发或cron触发。

### 产品评价

| 维度 | 评分 | 说明 |
|:-----|:-----|:-----|
| 方向性 | ⭐⭐⭐⭐⭐ | 多Agent协同是正确方向 |
| 完成度 | ⭐⭐⭐ | 还比较早期，Agent能力受限于底层模型 |
| 实用性 | ⭐⭐⭐ | 分析深度和精度还有很大提升空间 |
| 创新性 | ⭐⭐⭐⭐ | OpenClaw之后第一个把多Agent落到产品的 |
