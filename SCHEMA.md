# SCHEMA — 老H学金融知识库规范

> 本文档定义这个 Wiki 的组织结构、页面规范和 AI 操作行为准则。
> LLM 在创建、更新、链接任何页面时必须遵守本规范。

---

## 三层架构

```
老H学金融/
├── purpose.md          ← 知识库灵魂（目标、关键问题、研究范围）
├── SCHEMA.md           ← 本文件（组织规范）
├── raw/                ← 第一层：原始资料（只读）
│   ├── sources/        ← 原始文档存放处
│   └── inbox/          ← 待处理的新资料（临时）
├── docs/               ← 第二层：Wiki（AI 维护的知识网络）
│   ├── index.md      ← 知识库总索引
│   ├── log.md        ← 更新日志
│   ├── 01-基础理论/
│   ├── 02-交易系统/
│   ├── 03-实战复盘/
│   ├── 04-个股研究/
│   ├── 05-方法论/
│   └── 06-读书笔记/
└── mkdocs.yml          ← MkDocs 网站配置
```

### 层级职责

| 层级 | 内容 | 谁来维护 | 谁来读取 |
|------|------|----------|----------|
| **Raw Sources** | 原始文章、论文、研报、简报、网页剪藏 | 用户或 Cron 任务 | AI 只读不改 |
| **Wiki** | 结构化知识页（实体、概念、资料摘要、对比、综述） | **AI 全权维护** | 用户阅读 |
| **Schema** | 规范文件、目的文件 | 用户与 AI 共同迭代 | AI 阅读 |

## 页面类型

每个 Wiki 页面必须有明确的 `type`，写在 YAML frontmatter 中。

### 支持的 type

| type | 说明 | 存放位置 | 示例 |
|------|------|----------|------|
| `entity` | 实体 — 可指的事物 | `04-个股研究/` 或 `01-基础理论/` | 个股、人物、工具、公司 |
| `concept` | 概念 — 抽象思想或方法论 | `05-方法论/` 或 `02-交易系统/` | 回调vs退潮、主线识别、做T框架 |
| `source` | 资料摘要 — 对原始资料的摘要 | `03-实战复盘/` 或 `06-读书笔记/` | 每日复盘简报、论文摘要 |
| `comparison` | 对比分析 — 多个实体/概念的横评 | 对应主题目录 | 回调vs退潮、科创板vs主板 |
| `overview` | 综述 — 跨文档的综合分析 | `docs/` 根目录 | 市场阶段判断、板块轮动综述 |

### YAML Frontmatter 规范

每个 Wiki 页面必须包含以下 frontmatter：

```yaml
---
type: entity | concept | source | comparison | overview
title: "页面标题"
date: "YYYY-MM-DD"
source: "原始资料名或 URL（可选）"
sources: []
  - "raw/sources/文件名.md"
tags: []
  - "标签1"
  - "标签2"
status: active | archived | draft
---
```

- `sources: []` — 链接回原始资料文件，支持跟踪
- `tags: []` — 便于分类和检索
- `status` — `active` 活跃、`archived` 归档、`draft` 草稿

## 目录结构规范

### 01-基础理论

经济学、金融学、心理学等基础理论。

- 页面 type: `concept` 为主，少量 `entity`
- 例如：行为金融学、经济周期、量价关系、多元思维模型

### 02-交易系统

可复现的交易规则、策略和框架。

- 页面 type: `concept` 为主
- 例如：主线龙头战法、仓位管理、做 T 框架、止损止盈

### 03-实战复盘

每日大盘分析、操作记录、经验教训。

- 页面 type: `source` 为主（简报），结合 `overview`
- 命名规范：`YYYY-MM-DD-类型.md`
- 类型：早报、盘中分析、尾盘分析、收盘复盘、经验教训

### 04-个股研究

个股深度分析，包含基本面 + 技术面。

- 页面 type: `entity`
- 命名规范：`股票名称.md`
- 必须包含：公司业务、产品/客户、财务数据、技术面 K 线、操作路线图

### 05-方法论

具体的分析方法、判断框架、检查清单。

- 页面 type: `concept` 为主，少量 `comparison`
- 例如：五维校验法、123 冷冻期、主线识别三要素

### 06-读书笔记

金融经典阅读笔记。

- 页面 type: `source` 为主
- 例如：威科夫操盘法、股票大作手回忆录

## 双向链接规范

### 语法

使用 Obsidian `[[wikilink]]` 语法：

```markdown
在主线识别中，需要关注 [[量价关系]] 和 [[资金流向]] 两个维度。
```

### 链接规则

1. **必须存在对应页面** — 不要链接到不存在的页面
2. **新概念首次出现时创建页面** — 如果 `[[某概念]]` 尚不存在，创建一个草稿页面
3. **一个概念只对应一个页面** — 同一概念不要出现多个页面，避免重复
4. **充分链接** — 每个页面尽量包含相关的双向链接

### 链接类型

| 链接方式 | 语法 | 用途 |
|----------|------|------|
| 双向链接 | `[[页面名]]` | 知识网络内部关联 |
| 嵌入链接 | `[[页面名|显示文字]]` | 改变显示文字 |
| 标题链接 | `[[页面名#标题]]` | 链接到页面内的标题 |
| 外部链接 | `[text](URL)` | 网页、论文等外部资源 |

## AI 操作行为准则

### 两条铁律

1. **用户说"写入知识库"— AI 才操作 Wiki**
   - 接收文档/文本/URL
   - 自动分析 → 提取实体、概念 → 创建/更新页面 → 添加双向链接 → 更新索引
   - 更新 `log.md` 记录操作

2. **用户说"结合知识库"— AI 才检索 Wiki**
   - 搜索相关页面 → 综合分析 → 带引用地回答
   - 回答可以反向存回知识库（如果价值高）

### 日常对话

- **不主动读取知识库** — 普通问答不触发知识库检索
- **不主动写入知识库** — 不在未被要求的情况下修改 Wiki

### 避免污染

- 不将无关对话的内容写入知识库
- 不在知识库中存储临时性或私人信息
- 保持知识库的专业性和投资决策相关性

## 三大核心操作规范

### 1. Ingest（摄入）

**触发条件**：用户提供文档、URL 或说"把这篇写入知识库"

**步骤**：
1. 读取原始资料（保存到 `raw/sources/` 或 `raw/inbox/` 临时存放）
2. 分析：提取关键实体、核心概念、主要论点、与现有知识的关联
3. 生成资料摘要页（type: source）
4. 更新/创建相关实体页（type: entity）
5. 更新/创建相关概念页（type: concept）
6. 添加双向链接 `[[...]]`
7. 更新 `index.md` 索引
8. 记录到 `log.md`

**输出要求**：
- 每份资料至少生成一个摘要页
- 关联所有相关实体和概念
- 更新日志记录操作

### 2. Query（查询）

**触发条件**：用户说"结合知识库"或明确要求检索已有知识

**步骤**：
1. 分析问题，确定检索范围
2. 搜索相关页面（标题、内容、tags、双向链接）
3. 读取相关页面全文
4. 综合分析，带引用回答（格式：[1] [2]）
5. 如果回答价值高，询问是否存回知识库

**引用格式**：
```markdown
根据知识库中的分析 [1][2]，当前市场处于震荡期...

[1] [[回调vs退潮五维校验法]]
[2] [[混沌期生存法则]]
```

### 3. Lint（审查）

**触发条件**：用户说"审查知识库"或定期（每周一次）

**检查项**：
1. **矛盾检查** — 不同页面之间的冲突陈述
2. **过期检查** — 是否有明显过时的信息（如老黄已不再持有某股、市场阶段已变化）
3. **孤儿页面** — 没有任何入链的页面（度 = 0）
4. **缺失链接** — 应该有关联但未链接的页面
5. **重复页面** — 同一概念多个页面

**输出**：
- 生成审查报告（Markdown）
- 列出发现的问题 + 修复建议
- 如果是自动化的，直接修复可自动修复的问题

## 文件命名规范

### Wiki 页面

```
{date}-类型.md           ← 实战复盘类（如 2026-05-18-盘中分析.md）
{股票名称}.md            ← 个股研究（如 中兴通讯.md）
{概念名称}.md            ← 方法论/概念（如 回调vs退潮五维校验法.md）
{书名}或{论文名}.md     ← 读书笔记/资料摘要
```

### 原始资料

```
raw/sources/
  {YYYY-MM-DD}-{source_name}.md    ← 带日期的资料
  {source_name}.pdf                ← PDF 原文
  {source_name}.png/.jpg           ← 图片截图
```

```markdown
  - 引用：[[页面1]]、[[页面2]]
  - 反向存储：2026-05-18-问题分析.md
```

## 与 MkDocs 的兼容

- 所有 Wiki 内容在 `docs/` 目录下，MkDocs 可直接构建
- `raw/` 目录不在 MkDocs 构建范围内，保持独立
- 双向链接 `[[...]]` 在 MkDocs 中显示为普通文本，不影响渲染
- Obsidian 中 `[[...]]` 可点击跳转
